1. Why Health Analytics with Python?

Healthcare generates more data per person than almost any other industry — electronic health records, claims databases, imaging, genomics, wearables, and administrative files — yet much of it remains under-analysed. Python has become the language of choice for health data scientists because of its rich ecosystem spanning statistics, machine learning, natural language processing, geospatial analysis, and production deployment.

This series bridges three gaps that most textbooks leave open:

  1. Clinically meaningful examples — every dataset, model, and metric is grounded in real healthcare problems: readmission prediction, drug optimisation, spatial health inequities, causal effects of treatment.
  2. End-to-end thinking — notebooks do not stop at model accuracy. They continue through calibration, fairness, explainability, API serving, monitoring, and clinical governance.
  3. Reproducibility by design — code is written as it would be in a health system data science team: version-pinned environments, SHA-256 data manifests, logged pipelines, and CI/CD-ready test suites.

Who is this for? Health informatics professionals, clinical data scientists, epidemiologists, public health researchers, and graduate students who know Python basics and want to apply it rigorously to healthcare problems.

def sigmoid(x):

# ── Shared synthetic cohort ──────────────────────────────────────────────────
age       = np.random.normal(63, 14, N).clip(18, 90).astype(int)
los       = np.random.gamma(2.5, 1.8, N).clip(1, 28).astype(int)
hf        = np.random.binomial(1, 0.22, N)
dm        = np.random.binomial(1, 0.35, N)
copd      = np.random.binomial(1, 0.18, N)
payer     = np.random.choice(["Medicare","Medicaid","Private","Self-pay"],
                              N, p=[0.40,0.22,0.28,0.10])
logit     = -3.2 + 0.6*hf + 0.5*dm + 0.3*copd + 0.02*(age-63) + 0.2*(los>7)
readmit   = np.random.binomial(1, sigmoid(logit), N)
cvd_rate  = 180 + 1.4*np.random.normal(10,5,N) + 12*np.random.normal(0,1,N)

df = pd.DataFrame({
    "age": age, "los": los, "hf": hf, "dm": dm, "copd": copd,
    "payer": payer, "readmit": readmit, "cvd_rate": cvd_rate,
})

Reproducibility first - All notebooks use np.random.seed(42) as the first executable cell - Synthetic datasets are generated in-memory; no downloads required - A shared helper pattern regenerates the same cohort across all notebooks in a module

Code style - PEP 8 compliant, max line length 100 characters - Type hints in all function signatures - Docstrings on every function: purpose, args, returns - Tested with pytest; CI via GitHub Actions

Figures - All publication figures saved at 300 dpi via plt.savefig(..., dpi=300) - Consistent colour palette: #2563EB (blue) / #DC2626 (red) / #16A34A (green) - Colourblind-accessible palettes used throughout

6. Datasets & Data Ethics

Synthetic data — why not real patient data?

All notebooks use synthetic data generated with controlled random seeds. This approach:

  • Allows open publication without IRB restrictions or HIPAA concerns
  • Lets us embed known ground-truth effects so learners can verify their results
  • Reproduces identical results across operating systems and time

The synthetic data are designed to mimic the statistical properties of real EHR data (MIMIC-III), with realistic covariate distributions, correlation structures, and outcome rates.

7. Key Terms Quick Reference

Term Module Meaning
AUC-ROC 04 Area under the receiver-operating-characteristic curve; model discrimination
Brier score 04 Mean squared error of probability predictions; measures calibration
DCA 04 Decision Curve Analysis; net benefit of a model at a given risk threshold
SHAP 04 SHapley Additive exPlanations; feature contribution scores per patient
DAG 06 Directed Acyclic Graph; encodes causal assumptions about variable relationships
IPTW 06 Inverse Probability of Treatment Weighting; creates a pseudo-randomised sample
TMLE 06 Targeted Maximum Likelihood Estimation; doubly-robust causal effect estimator
DiD 06 Difference-in-Differences; compares treated vs control groups before/after a policy
ITS 06 Interrupted Time Series; evaluates policy effects using temporal trends
SMR 07 Standardised Mortality Ratio; observed / expected deaths in a population
LISA 07 Local Indicators of Spatial Association; identifies local spatial clusters
BYM 07 Besag-York-Mollié model; Bayesian disease mapping with CAR/ICAR prior
GWR 07 Geographically Weighted Regression; spatially varying coefficient model
2SFCA 07 Two-Step Floating Catchment Area; healthcare access metric
E-value 06 Minimum confounding effect size needed to explain away an observed association
HPSA 07 Health Professional Shortage Area (federal US designation)
PHI 05 Protected Health Information (HIPAA-regulated patient identifiers)
SaMD 08 Software as a Medical Device (FDA classification for clinical AI)
Model Card 08 Structured documentation of a model’s intended use, performance, and limitations
